Platform

All API settings that are platform-dependent must be adapted to the platform on which API is compiled/running.

This is done in VL53LX_platform.h file.

1. Sensor device type definition

User must provide an allocated instance of VL53LX_Dev_t type (in VL53LX_platform.h file) for each device he wants to use through

int VL53LX_xxxx(VL53LX_Dev_t dev, ... )

2. Device Read & Write access

API low-level functions rely on a few set of read & write functions which perform the access to the device. These functions must be implemented by the end user according to his platform configuration. Internal register access functions should be used :

  • VL53LX_WriteMulti()
  • VL53LX_ReadMulti()
  • VL53LX_WrByte()
  • VL53LX_WrWord()
  • VL53LX_WrDWord()
  • VL53LX_RdByte()
  • VL53LX_RdWord()
  • VL53LX_RdDWord()

    3. Data Types declaration

API functions rely on data types which are defined in VL53LX_types.h file. This file may require user attention and porting in case of warning messages.

4. Delay for polling operations

API polling high level functions do call the function VL53LX_PollingDelay() inside their while loop. A default implementation of the VL53LX_PollingDelay() function is provided. You may decide to change and implement your own VL53LX_PollingDelay() function.

5. API logging

All API functions entry and leave can be logged to help debugging issues. By default logging is disabled please define VL53LX_LOG_ENABLE at compilation level. If logging is enabled, a small set of macros must be implemented to adapt logging operation to the platform : _LOG_FUNCTION_START, _LOG_FUNCTION_END and _LOG_FUNCTION_END_FMT
